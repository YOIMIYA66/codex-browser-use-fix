开启 Codex Browser Use 插件

这是一份面向普通用户的 Windows 教程，用于在 Codex 桌面应用中启用本地自带的 Browser Use 插件，并处理“能看到插件但安装失败”的缓存问题。

适用范围
  • Codex 桌面应用已安装
  • 本地存在 openai-bundled 插件目录
  • 插件页显示 Browser Use 但安装失败

目录

1. 确认插件目录 2. 读取版本号 3. 注册 Marketplace 4. 开启功能开关 5. 启用插件配置 6. 修复安装缓存 7. 验证结果 8. 重启应用 常见问题
1

确认本机是否存在 browser-use 插件

先确认你的 Codex 桌面应用安装目录里是否有下面这个目录。没有这个目录的话，后续步骤无法继续。

目标目录格式
<Codex安装目录>\app\resources\plugins\openai-bundled\plugins\browser-use

Windows 上可能类似下面这种形式，版本号和包 ID 因机器而异：

示例路径
<Codex安装目录>\app\resources\plugins\openai-bundled\plugins\browser-use

使用 PowerShell 检查：

PowerShell
$browserUsePath = "<Codex安装目录>\app\resources\plugins\openai-bundled\plugins\browser-use"
Test-Path $browserUsePath
返回 True 表示本地确实带有 Browser Use 插件源码，可以继续。
2

读取插件版本号

Codex 插件缓存目录需要使用插件元数据里的版本号。先读取：

PowerShell
Get-Content "$browserUsePath\.codex-plugin\plugin.json"

找到类似内容：

plugin.json
{
  "name": "browser-use",
  "version": "0.1.0-alpha1"
}
本文后续用 0.1.0-alpha1 举例。如果你的插件版本不同，请使用你自己的版本号。
3

注册 openai-bundled Marketplace

找到 bundled marketplace 根目录：

目录格式
<Codex安装目录>\app\resources\plugins\openai-bundled

执行注册命令：

PowerShell
$bundledMarketplace = "<Codex安装目录>\app\resources\plugins\openai-bundled"
codex plugin marketplace add $bundledMarketplace
新版 Codex CLI 使用 codex plugin marketplace add。部分旧教程里的 codex marketplace add 可能已经不适用于新版本。
4

开启 Browser Use 依赖的功能开关

Browser Use 需要远程控制能力：

PowerShell
codex features enable remote_control

检查相关功能状态：

PowerShell
codex features list | Select-String -Pattern "remote_control|browser_use|in_app_browser|computer_use|plugins"

理想情况下，下面这些项都应该是 true：

  • browser_use
  • computer_use
  • in_app_browser
  • plugins
  • remote_control
5

在 config.toml 中显式启用插件

打开 Codex 配置文件：

PowerShell
notepad "$env:USERPROFILE\.codex\config.toml"

添加：

config.toml
[plugins."browser-use@openai-bundled"]
enabled = true

同时确认配置里已经有类似内容：

config.toml
[marketplaces.openai-bundled]
source_type = "local"
source = "<你的 openai-bundled 路径>"

[features]
remote_control = true
6

如果点击安装失败，手动补齐插件缓存

有些 Windows 环境中，插件页能看到 Browser Use，但点击安装会失败。常见原因是 marketplace 已注册，但用户目录下的插件 cache 没有正确生成。

正常 cache 结构应该是：

目录格式
%USERPROFILE%\.codex\plugins\cache\openai-bundled\browser-use\<版本号>

手动复制脚本如下：

PowerShell
$src = "<Codex安装目录>\app\resources\plugins\openai-bundled\plugins\browser-use"
$version = "0.1.0-alpha1"
$dst = "$env:USERPROFILE\.codex\plugins\cache\openai-bundled\browser-use\$version"

New-Item -ItemType Directory -Force -Path $dst | Out-Null

$srcRoot = (Resolve-Path -LiteralPath $src).Path.TrimEnd('\')
$dstRoot = (Resolve-Path -LiteralPath $dst).Path.TrimEnd('\')

Get-ChildItem -LiteralPath $srcRoot -Force -Recurse -Directory | ForEach-Object {
  $rel = $_.FullName.Substring($srcRoot.Length).TrimStart('\')
  New-Item -ItemType Directory -Force -Path (Join-Path $dstRoot $rel) | Out-Null
}

Get-ChildItem -LiteralPath $srcRoot -Force -Recurse -File | ForEach-Object {
  $rel = $_.FullName.Substring($srcRoot.Length).TrimStart('\')
  $target = Join-Path $dstRoot $rel
  New-Item -ItemType Directory -Force -Path (Split-Path -Parent $target) | Out-Null
  [System.IO.File]::WriteAllBytes($target, [System.IO.File]::ReadAllBytes($_.FullName))
}
这里没有使用普通的 Copy-Item -Recurse，是为了避开部分 Windows 安装目录上的特殊文件属性导致的复制失败。
7

验证插件是否可被 Codex 识别

先检查关键文件：

PowerShell
Test-Path "$env:USERPROFILE\.codex\plugins\cache\openai-bundled\browser-use\0.1.0-alpha1\.codex-plugin\plugin.json"

返回 True 后，再检查 Codex 是否能加载：

PowerShell
codex debug prompt-input "test" | Select-String -Pattern "browser-use|Browser Use"

如果看到类似内容，说明已成功：

期望输出片段
browser-use:browser
Browser Use
.codex\plugins\cache\openai-bundled\browser-use\0.1.0-alpha1
8

重启 Codex 桌面应用

最后彻底退出并重启 Codex 桌面应用。重启后，Browser Use 应该已经可用。

如果插件页仍显示未安装，但 codex debug prompt-input 已能看到 Browser Use，通常说明功能实际已经加载，不必重复点击安装。

常见问题

为什么版本号是 0.1.0-alpha1？

因为插件自己的 .codex-plugin\plugin.json 里写的就是这个版本。Codex 插件缓存目录按 <marketplace>\<plugin>\<version> 组织，所以目录名必须和插件元数据一致。

为什么 UI 能看到插件，但安装失败？

通常是 marketplace 注册成功了，但插件文件没有成功复制到用户目录下的 .codex\plugins\cache。手动补齐 cache 后即可解决。

这份教程是否包含个人信息？

不包含。路径里的版本号、安装目录和包 ID 都用占位符表示，用户需要替换为自己机器上的实际路径。

教程基于 Windows + Codex Desktop 环境整理。执行前请确认路径与版本号来自你自己的安装目录。
